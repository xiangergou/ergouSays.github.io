什么是单例？

保证一个类仅有一个实例，并提供一个访问他的全局访问点。

曾探javascript设计模式与探索实践
单例模式是一种十分简单而且能够很简单的方式实现的一种设计模式。它是指在一个类中只有一个实例，即使你多次实例化这个类，它也只会返回一次实例化的对象，可以有效地避免资源的浪费，能减少不必要的内存开销, 并且在减少全局的函数和变量冲突也具有重要的意义。

“单例实现的方法一般是先判断实例存在与否，如果存在直接返回，如果不存在就创建了再返回，这就确保了一个类只有一个实例对象。在JavaScript里，单例作为一个命名空间提供者，从全局命名空间里提供一个唯一的访问点来访问该对象。”
——汤姆大叔的博客 深入理解javascript系列(25)



实现方式

  • 最简单的实现方式
    在这之前，且让我们牢记单例的核心：
    1. 确保只有一个实例。
    2. 提供全局访问。

看官是否有了想法了？

timeTool独一无二，放在全局作用域下，我们可以在任意位置调用它。这已然满足了单例模式的条件。
然而事实上，全局变量不算单例模式，只是在js开发中，我们将其当做单例模式来使用。 全局变量存在诸多问题，最主要的当属命名空间污染。 js创造者Brendan Eich也曾名言全局变量是设计上的失误。

最常见的情况是，项目由两个人合作，若都不注意全局污染，就容易出现变量重复声明的情况。


  • 使用命名空间
    适当的使用命名空间，不会杜绝全局变量，但可以减少其数量。比如 
    // 开发者A
let devA = {
  name: function() {  
      alert('a')
  }
}
// 开发者B
let devB = {
  name: function() { 
      alert('b')
  }
}
