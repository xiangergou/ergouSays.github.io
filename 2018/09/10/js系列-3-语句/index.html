Posted on 2018-09-10 | In js深入理解系列
Words count in article: 1,610 字 | Reading time ≈ 7 分钟


语句相当于句子，表达式相当于短语，运算符(操作符)则相当于标点符号和连接词。

语句的结果值

语句都有一个结果值(statement completion value，undefined 也算)。 获得结果值最直接的方法是在浏览器开发控制台中输入语句，默认情况下控制台会显示所执行的最后一条语句的结值。

复合语句和空语句

花括号将语句括起来即为复合语句

语句结尾无需分号，
块中语句需分号。

声明语句

var： 无法通过delete删除。 变量提升。
function: 函数定义不能出现在if或者while中。 函数提升。 函数不能删除，但能重写。

if语句

if语句后常跟else if， 事实上 JavaScript 没有 else if， 我们经常用到的else if实际上是这样的:

if (b) { .. } else { .. }，实际上，跟在else后面的一个单独的语句，带不带{ }都可以。

do while语句

后测试循环语句，表示循环体的代码最少被执行一次。

while语句

若控制表达式恒为true，将创建无线循环。

for语句

前测试语句。
for循环中，省略初始化表达式、控制表达式、循环后表达式，将创建无限循环。

for in语句

枚举对象属性。
for in循环输出的对象属性无序。

关于代码块，还有些坑。

表面上看 + 运算符根据第一个操作数( [] 或 {} )的不同会产生不同的结果，实则不然。 第一行代码中，{} 出现在 + 运算符表达式中，因此它被当作一个值(空对象)来处理。[] 会被强制类型转换为””，而 {} 会被强制类型转换为”[object Object]”。
但在第二行代码中，{} 被当作一个独立的空代码块(不执行任何操作)。代码块结尾不需要分号，所以这里不存在语法上的问题。最后 + []将[]显式强制类型转换为 0 。

break 和 continue语句

break和continue通常和标签页一起使用，前者表示跳出当前循环进入标签标示的语句块的结束，或直接闭合这个语句块的执行。 而continue表示转而执行标签页循环。

throw语句

当程序出现错误或使用throw语句就会显示的抛出异常。
经常使用Error类型及其子类型。 一个Error对象有name属性表示错误类型，一个message用来存放传递给构造函数的字符串。

当抛出异常时，js停止当前逻辑，并跳转至最近的异常处理程序。 与作用域链类似，寻找异常处理程序也是向上传播。

try/catch/finally语句

try/catch/finally是js的异常处理语句，try定义需要处理的异常所在的代码块。
finally语句定义与promise的finally类似，无论程序出错与否，都会执行finally语句。

catch语句参数e表try抛出的error对象。 若try报错，且不存在处理此错误的catch语句，则解释器会先处理finally语句，之后向上查找。

try 中有 return 语句会出现什么情况?

return 42 先执行，并将 foo() 函数的返回值设置为 42。然后 try 执行完毕，接着执行 finally。最后 foo() 函数执行完毕，console.log(..) 显示返回值。
