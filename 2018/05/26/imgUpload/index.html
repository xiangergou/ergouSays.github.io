需求分析： 我们需要图片批量上传，图片预览、可选择性的编辑(裁剪)。

1. 图片上传

我们知道传统图片上传展示一般都是图片上传后返回地址，前端再由地址进行展示。
而此次我们采用createObjectURL将图片进行本地展示，待最终的调整后再进行上传的方式。
关于createObjectURL

step one:

multiple字段支持多个值的文件上传，我们用以实现图片多张上传。

2. 图片预览

大多数情况下的图片预览插件vue2-preview足以满足要求，然我们这基础上添加编辑功能，这就意味着我们要参照PhotoSwipe作出修改。

我们采用extend创建vue“子类”的形式完成我们的priview功能。

结构如下:
-Preview
–plugins
—index.js
—preview.vue

我们在preview.vue文件下添加我们的photoswipe。

我们在页面收尾出添加“编辑”button,以作为裁剪功能的入口。
此处注意，当对photoswipe添加自定义方法时，需要在data()中添加“isClickableElement: function (el) {
return /edit_img/.test(el.className)
}”, 解除其对自定义的限制，否则自定义事件是无法生效的。
在methods中，我们申明包括’open’、’close’以及我们自定义的’previewEdit’方法。
previewEdit向父组件发出“编辑”命令通知。父组件得到通知后，关闭预览组件并调用裁剪组件。
